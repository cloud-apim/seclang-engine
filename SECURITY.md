# Security Policy

We strongly advise you to update your scala-seclang-engine usage as soon as a new version is released.

## Supported Versions

Each version is supported until the next one is released

## Reporting a Vulnerability

We want to keep scala-seclang-engine safe for everyone. If you've discovered a security vulnerability in scala-seclang-engine, we appreciate your help in disclosing it to us in a responsible manner by using the dedicated [github form](https://github.com/cloud-apim/scala-seclang-engine/security) or by sending an email to [security@cloud-apim.com](mailto:security@cloud-apim.com) containing your name, an email address, a description of the vulnerability, the impacted versions of scala-seclang-engine. You can encrypt the email content using [this pgp key](https://mathieuancelin.keybase.pub/pgp_key.asc)
